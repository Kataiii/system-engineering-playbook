# Введение

## Назначение

Назначение данного документа для разработки мобильного приложения с
ИИ-тренером, предназначенного для автоматизации управления тренировочным
процессом пользователя. Система обеспечивает персонализацию и повышение
эффективности тренировок за счет непрерывного анализа данных и адаптации
нагрузок.

## Термины:

## Общие термины

**ВИ** -- Вариант Использования или Use Case.

**ПО** - программное обеспечение.

**МП** - мобильное приложение.

**Устройство -** мобильный телефон на котором происходит работа
разрабатываемого приложения.

**Система** - мп дневника тренировок с ИИ-тренером для умного
планирования, адаптации нагрузок и мотивационной поддержки.

**ОС** - операционная система.

## Бизнес термины

**Тренировочный процесс -** комплекс тренировок, отделенных определеным
промежутком времени друг от друга, позволяющих при полном выполнении
использовать все группы мышц.

**Тр** - тренировка.

**Сплит** - тренировочный процесс делится на дни, посвященные разным
мышечным группам.

**Фуллбади** - тренировочный процесс предусматривает использование всех
групп мышц в рамках одной тренировки

**Круговая тренировка** - тренировка, состоящая из последовательности
упражнений (станций), выполняемых одно за другим с минимальным отдыхом.
После завершения круга следует отдых, и цикл повторяется.

**Суперсет** - Последовательное выполнение двух упражнений без отдыха
между ними.

**Трисет -** Последовательное выполнение трех упражнений без отдыха
между ними.

**Гигантский сет** - Последовательное выполнение четырех и более
упражнений без отдыха.

**Дроп-сет** - Метод повышения интенсивности, при котором после отказа в
упражнении вес сразу уменьшается (на 20-25%), и подход продолжается без
полноценного отдыха. Может быть двойным, тройным и т.д.

**Рабочий вес** - Вес отягощения, используемый для выполнения
запланированных подходов и повторений.

**Повторение** - Одно полное движение упражнения от начальной до
конечной точки и обратно.

**Подход** - Непрерывная серия повторений одного упражнения.

**Целевая мышца** - Основная мышечная группа, на которую направлено
упражнение.

**Мышцы-синергисты** - Мышцы, которые помогают целевой мышце в
выполнении движения.

**Мышцы-стабилизаторы** - Мышцы, которые фиксируют положение суставов и
тела во время упражнения.

**Мышцы-антагонисты** - это пары мышечных групп, которые выполняют
противоположные движения вокруг одного и того же сустава, например,
сгибание и разгибание. Например, бицепс сгибает локоть, а трицепс ---
разгибает

**Альтернативное упражнение -** схожее упражнение с другим, с которым
оно будет чередоваться в рамках выполнения тренировки.

## Технические термины

**ИИ** - искусственный интеллект.

**API** - Набор правил и методов, с помощью которых фронтенд
(приложение) взаимодействует с бэкендом.

**Оффлайн-режим** - Возможность работы с приложением (просмотр планов,
запись выполненных упражнений) без подключения к интернету с последующей
синхронизацией.

**SDK** - Набор инструментов от стороннего поставщика для интеграции его
услуг в ваше приложение.

**Push-уведомления** - Сообщения, которые приходят на устройство даже
когда приложение не запущено.

**Тост** - всплывающее сверху экрана на всю его ширину, небольшое
текстовое сообщение с возможность его закрыть нажатием на крестик, либо
свайпом всторону.

## Обзор продукта

Система представляет собой мобильное приложение-дневник тренировок,
ключевой особенностью которого является встроенный ИИ-тренер. Продукт
функционирует как автономный цифровой помощник, автоматизирующий процесс
планирования, выполнения и адаптации фитнес-нагрузок. Основная ценность
системы заключается в способности анализировать пользовательские данные
и генерировать персонализированные тренировочные программы, обеспечивая
тем самым индивидуальный подход к каждому пользователю.

Приложение является самодостаточным решением и для базовой работы не
требует интеграции со сторонними системами. Для расширения
функциональности возможна опциональная синхронизация с сервисами
здоровья (Apple HealthKit, Google Fit) для автоматического сбора
показателей.

## Функции продукта

Ключевые функции системы включают:

- Создание тренировки (создать шаблон тренировку с возможность
  добавлять в неё упражнение)

- Создание тренировки по советам ИИ-тренера

- Фиксация повторений (зафиксировать кол-во повторений для подхода, с
  дальнейшим пересчетом рекомендованных повторений для дальнейших
  подходов)

- Просмотр созданных тренировок (просмотр календаря тренировок, с
  указанными в них рекомендованным кол-во повторений на подход в
  упражнении)

- Редактирование тренировок

- Поиск упражнений (поиск в общем списке, по названию, также с
  возможностью фиксации, также приложение должно исеть возможность
  выдать рекомендованные упражнения)

- Фильтрация упражнений (по группам мышц, по мышцам-фиксатора и т.д)

- Умный подбор упражнений (на мышцы-антогонисты)

- Просмотр / получние мотивирующих сообщений от тренера при достижении
  успехов

- Просмотр / получение выговоров от тренера при пропуске тернировок

## Характеристики пользователей

2.  Любители - регулярно занимающиеся спортом пользователи со средним

3.  Продвинутые - опытные спортсмены, требующие сложных программ и
    детальной аналитики, прогрессии и т.д.

# Функциональные требования

## Вариант использования «Создание тренировки через ИИ-тренера»

**Цель:** Получить и подтвердить персонализированный план тренировки,
автоматически сгенерированный ИИ на основе данных пользователя. На
выходе получаем шаблон тренировки/тренировочного процесса.

**Предусловие:** Нет

**Основной поток:**

1.  Пользователь перешел к этапу создания тренировки / тренировочной
    программы через ИИ-тренера

2.  Система из профиля получает необходимые данные для более корректного
    подбора упражнений

3.  Система уточняет у пользователя варианты создания (отдельная
    тренировка, тренировочная программа)

4.  Пользователь вносит данные о желаемой тренировки (продолжительность,
    тип тренировки (сплит, фуллбоди), место тренировки (дом, улица,
    зал), цель тренировки (массонабор, похудение, поддержание в тонусе),
    сущетсвующий шаблон тенировки (опционально)), если пользователь до
    этого не создавал тренировки через ИИ-тренера, иначе система
    показывает уже предзаполненную форму

5.  Система генерирует тренировку/тренировочный процесс, с возможность
    «Отменить»

6.  После успешной генерации, у пользователя есть возможность, изменить
    вручную тренировку / тренировочную программу,подтвердить, закрыть
    созданное.

7.  После подтверждения тренировки добавляются в личное расписание.

    **Альтернативные потоки:**

    **Пользователь не заполнил данные в профиле:**

8.  На этапе 3.1.2 в профиле не оказалось нужных данных, либо есть
    данные заполнены не полностью.

9.  Если обязательные данные заполнены, система показывает окно, с
    предложением дозаполнить данные для более правильного создания
    тренировки. Окно возможно закрыть и перейти в п. 3.1.3 основного
    потока.

10. Основные данные незаполнены, система показывает окно без возможности
    продолжения основного потока, с возможность нажать кнопку «Отмена»,
    если данные заполнены, они запоминаются в профиль, окно закрывается,
    происходит переход в п. 3.1.3 основного потока.

11. При нажатии на кнопку «Отмена», сценарий создания тренировки через
    ИИ заканчивается.

    **Пользователь отменяет генерацию:**

12. На этапе 3.1.5. при нажатии на кнопку «Отменить», либо при нажатии
    на системную кнопку «Назад» система спрашивает подверждение.

13. При подтвеждении генерация заканчивается, происходит переход на этап
    3.1.4. основного потока.

    **Пользователь добавляет тренировку в календарь:**

14. На этапе 3.1.7. если пользователь создал тренировочный процесс, то
    система предлагает дни недели выполнения отдельных тренировок
    (например нечетные дни недели или четные)

15. Также пользователю предоставляет возможность добавить тренировку(и)
    в Google calendar, системное приложение календаря.

16. При отмене происходит выход из сценария

17. При добавлении, появляется индикация добавления в календарь

18. При ошибке на этапе 3.1.19, показывается тост с ошибкой, происходит
    выход из сценария.

19. При успешном добавлении, показывается тост с подтверждением
    синхронизации календарей, происходит выход из сценария.

    **Ошибка при получении данных профиля:**

20. На этапе 3.1.2 происходит ошибка получения данных

21. Показывается тост с тексом ошибки

22. Переход к альтернативному сценарию **«Пользователь не заполнил
    данные в профиле»**

    **Ошибка при генерации тренировки /тренировочного процесса:**

23. На этапе 3.1.5. происходит ошибка

24. Система показывает тост с ошибкой и переводит на п. 3.1.4. основного
    сценария.

    **У пользователя есть активный тренировочный процесс:**

25. На этапе 3.1.7, если у пользователя уже есть активный тренировочный
    процесс.

26. Если пользователь создал новый тренировочный процесс, то система
    спрашивает желает ли он заменить текущий на новой, с вариантами
    «Нет», «Да».

27. При нажатии на «Нет», новый тренировочный процесс добавляется в
    резерв, в календаре не происходит никаких изменений. Основной
    сценарий заканчивается.

28. При нажатии на «Да», новый тренировочный процесс становится
    активным, просиходит переход на сценарий **«Пользователь добавляет
    тренировку в календарь»**

29. Если пользователь создал отдельную тренировку, то система спрашивает
    хочет ли он изменить существующий тренировочный процесс или добавить
    тренировку в резерв.

30. При добавлении в резерв происходит выход из сценария, календарь не
    изменяется.

31. При изменение тренировочного процесса происходит переход в **ВИ
    «Изменение тренировочного процесса».**

    **Бизнес-правила:**

32. Для генерации релевантного плана ИИ-тренер требует минимум 5
    заполненных полей профиля (опыт, вес, рост).

33. Пользователь может иметь не более 3 тренировочных программ
    одновременно, включая активную.

34. В тренировочной программе не может быть более 7 тренировок.

35. Тренировочная программа не должна быть только на одну группу мышц.

## Вариант использования «Изменение тренировочного процесса»

**Цель:** Изменить тренировочный процесс

**Предусловие:** У пользователя есть тренировочный процесс.

**Основной поток:**

36. Пользователю отображаются тренировки в рамках текущего
    тренировочного процесса, а также возможность «Отменить»,
    «Сохранить», а также добавить, заменить, изменить, удалить
    тренировку, изменить порядок, добавить в календарь (Google,
    системный)

37. При нажатии на «Отменить» просиходит завершение сценария.

38. При попытке заменить тренировку, отображаются окно с наиболее
    подходящими тренировками на замену.

39. При нажатии на тренировку, она выделяется и предлагается возможность
    подтведить выбор, либо закрыть окно.

40. При подтвеждении тренировка заменяется.

41. При попытке удалить тренировку, необходимо подтвердить удаление.

42. При добавлении тренировки, предоставляется список тренировок,
    ранжированных от наиболее подходящих к менее подходящим.

43. При выборе, необходимо подвердить, после чего тренировка добавляется
    в тр. процесс.

44. При попытке изменить порядок тренировок, происходит синхронизация с
    календарем (при наличии интеграции)

45. При ошибке синхронизации, пользователю показывается тост с
    соответствующей ошибкой.

46. При попытке добавить в календарь, текущий личный календарь тр.
    процесса добавляется в календарь (Google, системный)

    **Альтернативные потоки:**

    **У пользователя при отмене были изменения**

47. На этапе 3.2.2. при использовании «Отменить», отображается окно с
    подтвежлением отмены.

48. При подтвеждении происходит завершение сценария.

49. При отменени подтвеждения просиходит возвращение к основному потоку.

    **У пользователя нет подходящих тренировок на замену.**

50. Вместо функции «Заменить», появлется функция «Создать замену»

51. При использовании функции просиходит переход на **ВИ «Создание
    тренировки через ИИ-тренера»** c предвыбранной пунктом «тренировка»
    на этапе 3.1.3 (без возможности изменения), и с предзаполненными
    данными на этапе 3.1.4.

52. При ошибке генерации тренировки выводится тост с ошибкой и
    просиходит переходит переход к основному сценарию.

53. При успешной генерации просходит переход на основной сценарий,
    тренировка помечается предвыбранной и происходит переход на п. 3.2.4
    основного потока.

    **Пользователь удаляет последюю тренировку в тр. процессе.**

54. На этапе 3.2.6 в подтвеждении указывается, что удаление тренировки
    ведет к удалению тр. процесса.

55. При подверждении тр. процесс удаляется, сценарий завершается

56. При отмене, происходит переход к основному сценарию.

    **У пользователя нет тренировок в резерве, либо все тренировки
    являеются неудовлетворительными для тр. процесса.**

57. На этапе 3.2.7. если есть возможность создать подходящую тренировку,

58. При нажатии происходит переход на **ВИ «Создание тренировки через
